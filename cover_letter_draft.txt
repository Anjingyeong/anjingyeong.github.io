1. 자기소개 및 성장 과정
[AI/ML의 깊이와 풀스택 웹 인프라의 넓이를 연결하다]
소프트웨어 엔지니어링의 본질은 데이터가 가진 잠재력을 안정적인 인프라를 통해 실제 사용자 가치로 치환하는 것이라 믿습니다. 과거 컴퓨터 비전(VAE, RF-DETR) 연구를 수행하며 뛰어난 성능의 AI를 개발하더라도, 이를 흔들림 없이 감당할 수 있는 대규모 서비스 아키텍처가 없다면 단순한 '연구용 데모'에 머무른다는 한계를 뼈저리게 체감했습니다. 

이러한 갈증은 저를 '데이터 중심 AI 설계'를 넘어 'Full-Stack 웹 인프라 엔지니어'로 확장시키는 결정적 계기가 되었습니다. 프론트엔드의 UI/UX 상태 관리부터 백엔드의 트래픽 제어, 복잡한 데이터 파이프라인 연동에 이르기까지 전체 아키텍처의 병목 현상을 꿰뚫어 보는 시야를 기를 수 있었습니다. 딥러닝 텐서의 저수준(Low-level) 연산 로직을 뜯어보던 집요함을 바탕으로, 이제는 예외 상황에서도 멈추지 않는 가장 견고한 프로덕션 웹 서비스를 설계하는 데이터 기반 풀스택 엔지니어로 성장했습니다.


2. 지원동기 및 입사 후 포부
[트래픽 스파이크와 타임아웃 오류를 제어하는 실전형 아키텍트]
아무리 정교한 비즈니스 데이터라도 트래픽 스파이크 시 발생하는 DB 병목이나 동기식 렌더링 지연을 견디지 못하면 치명적인 장애로 이어집니다. 저는 FullCount(실시간 커뮤니티)와 Lumina Capital(금융 대시보드) 프로젝트를 통해 이러한 인프라 한계를 단독 트러블슈팅하여 타파해 낸 경험이 있습니다.

단방향 API Polling으로 인해 발생한 막대한 DB I/O 오버헤드를 WebSockets 및 Redis Pub/Sub 기반 아키텍처로 전면 교체하여 시스템 지연을 없앴으며, 동기식 데이터 스크래핑이 유발하는 타임아웃 붕괴를 비동기(Asynchronous) CRON 캐시 파이프라인으로 재조립해 렌더 딜레이를 0%에 가깝게 근절했습니다. 나아가 현재 SK 쉴더스 엔터프라이즈 과정을 이수하며 대용량 트래픽 통제 역량을 고도화하고 있습니다. 입사 후, 현장에 산재한 데이터 흐름의 병목을 가장 먼저 찾아내어 서빙 인프라와 배포 자동화 파이프라인을 극한으로 최적화해 내겠습니다.


3. 직무 관련 핵심 역량
[인프라 부하 80% 절감 및 실시간 추론 튜닝 등 한계를 부수는 최적화 역량]
저의 가장 뛰어난 무기는 제한된 하드웨어 리소스와 악조건 속에서도 최상의 성능을 끌어내는 '공학적 병목 최적화' 능력입니다. 

첫째, 대규모 상태 제어 및 백엔드 최적화 능력입니다. 에스크로 트랜잭션과 실시간 멀티플렉싱 통신이 혼재된 복잡한 환경에서, Spring Boot와 JPA 영속성 컨텍스트(Persistence Context)를 연계한 릴레이션 상태 머신을 구축하고 30초 지연 분산 오프로딩 체계를 적용하여 데이터 무결성을 유지한 채 DB 부하를 80% 이상 혁신적으로 절감했습니다.
둘째, 데이터 중심(Data-Centric) 튜닝 능력입니다. 의료 도메인의 극단적 데이터 한계 속에서 상용 엣지(Edge) 인프라 전개를 목표로, RF-DETR 아키텍처에 기하학적 왜곡(Elastic Deform) 증강 파이프라인을 직접 이식했습니다. 그 결과 mAP 7% 향상과 더불어 하드웨어 제약을 뚫고 22 FPS 이상의 실시간 추론 스루풋(Throughput)을 확보하며 캡스톤 공학혁신 대상을 수상했습니다.


4. 성격의 장단점
['프레임워크 블랙박스'를 해체하는 집요함, 그리고 MVP 기반의 실용적 타협]
눈앞에 출력되는 단순 에러 로그나 높은 모델 평가 수치에 안주하지 않고, 프레임워크 이면의 로우레벨(Low-level) 본질까지 끝까지 추적하는 집요함이 제 엔진입니다. 유방암 검출 비지도학습 AI 설계 당시, 기존 라이브러리에 의존하는 블랙박스 구조의 한계를 넘기 위해 TensorFlow 저수준 API를 직접 통제했습니다. KLD(분포 오차)와 MSE(재구성 오차)의 텐서 가중치를 조작하는 커스텀 손실 함수(β-VAE)를 밑단부터 재설계하여 암세포의 픽셀 이질감을 돋보이게 유도하고 실제 탐지력을 극한으로 끌어올렸습니다.

반면, 이러한 엔지니어링적 집요함 탓에 프로젝트 초기 구조 설계와 방어 로직에 과도한 시간을 쏟는 완벽주의 성향이 있었습니다. 저는 이 단점을 인지하고 비즈니스의 생명력인 빠른 검증(Time-to-market)을 위해 'MVP(최소 기능 제품) 우선 배포'라는 확고한 실무 원칙을 세웠습니다. 무결한 코드를 처음부터 고집하기보다, 핵심 비즈니스 아키텍처가 동작하는 프로토타입을 빠르게 배포한 뒤 실제 모니터링을 거치며 점진적으로 리팩토링합니다. 코드의 신뢰성을 절대 타협하지 않으면서도, 기획 및 비즈니스 파트와 가장 빠르고 민첩하게 호흡하는 유연한 풀스택 엔지니어로 헌신하겠습니다.
