# 자기소개서 - 안진경

## 1. 자기소개 및 성장과정

건양대학교 의공학과에서 AI와 데이터 분석을 전공하며, **의료 영상 분석부터 통신 네트워크까지** 다양한 분야의 AI 응용 기술을 경험해왔습니다.

대학 과정 동안 단순히 이론을 배우는 것에 그치지 않고, **실제 문제를 해결하는 AI 시스템 개발**에 집중했습니다. Transformer 기반 RF-DETR 모델로 대장 내시경 영상에서 용종을 실시간으로 검출하는 시스템을 개발하면서, 베이스라인 대비 mAP 7% 향상과 22 FPS의 처리 속도를 달성했습니다. 이 프로젝트로 성균관대학교 컨소시엄 창의적 종합설계 경진대회에서 동상을, 건양대학교 2025 캡스톤 경진대회에서 금상을 수상했습니다.

또한 VAE 기반 유방암 병변 검출 알고리즘 개발 프로젝트에서는 비지도 학습 방식을 적용해 Dice coefficient 40-90%의 성능을 달성하며 공학혁신상을 수상했습니다. 이러한 경험을 통해 **복잡한 문제를 구조화하고, 데이터 기반으로 해결하는 능력**을 키웠습니다.

충남경제진흥원 미래내일 일경험 인턴십에서는 경영 사무 업무를 담당하며 **조직 내 효율적인 커뮤니케이션과 프로세스의 중요성**을 배웠고, Lab-CORPS 산학협력 프로젝트를 통해서는 실제 산업 현장의 문제를 이해하고 기술적으로 해결하는 경험을 쌓았습니다.


## 2. 지원동기 및 입사 후 포부

**[회사명]**에 지원하게 된 이유는 **AI 기술을 통신 네트워크에 적용하여 실질적인 가치를 창출하고 싶다**는 목표 때문입니다.

의료 영상 분석 프로젝트를 진행하며 실시간 처리와 모델 경량화의 중요성을 깊이 이해하게 되었습니다. 이러한 경험은 통신 네트워크 환경에서도 동일하게 적용됩니다. 5G/6G 네트워크에서 발생하는 대용량 데이터를 실시간으로 처리하고, 지능형 트래픽 관리를 통해 네트워크 성능을 최적화하는 것은 제가 익혀온  **실시간 AI 모델 개발 및 최적화 경험**과 직접적으로 연결됩니다.

특히 제가 개발한 RF-DETR 시스템에서 Weight Pruning과 Query 최적화를 통해 정확도 손실 없이 연산 속도를 높인 경험은, 제한된 네트워크 환경에서도 효율적으로 작동하는 Edge AI 시스템 개발에 활용될 수 있다고 생각합니다.

입사 후에는 **통신 네트워크 최적화를 위한 AI 솔루션 개발**에 기여하고 싶습니다. 단기적으로는 네트워크 트래픽 패턴 분석, 이상 탐지, QoS 최적화 등 실무 경험을 쌓으며, 중장기적으로는 6G 통신에서 요구되는 **초저지연, 초연결 환경을 위한 지능형 네트워크 관리 시스템**을 연구·개발하는 엔지니어로 성장하고 싶습니다.


## 3. 직무 관련 경험 및 역량

### 실시간 AI 시스템 개발 역량

**RF-DETR 기반 실시간 대장 내 용종 검출 시스템 개발** 프로젝트에서 Transformer 기반 모델을 의료 영상에 최적화했습니다. 의료 데이터의 부족과 클래스 불균형 문제를 Elastic Deformation, Grid Distortion 등 고급 데이터 증강 기법으로 해결했고, Weight Pruning과 Query 수 최적화를 통해 경량화를 구현했습니다. 그 결과 베이스라인 대비 mAP 7% 향상과 22 FPS의 실시간 처리 속도를 달성했습니다.

이 경험을 통해 **실시간성이 중요한 시스템에서 정확도와 속도의 트레이드오프를 균형있게 조정하는 능력**을 키웠으며, 이는 통신 네트워크의 실시간 모니터링 및 최적화에 직접 적용 가능합니다.

### 데이터 기반 문제 해결 능력

**VAE 기반 유방암 병변 검출 알고리즘 개발** 프로젝트에서는 정상 데이터만으로 학습한 모델이 비정상 데이터를 검출하는 비지도 학습 방식을 구현했습니다. 동적 임계값 이진화 기법을 도입하여 Dice coefficient 40-90%의 성능을 달성했습니다.

이처럼 **제한된 데이터 환경에서도 창의적인 접근으로 문제를 해결하는 능력**은, 통신 네트워크에서 라벨링 되지 않은 대량의 로그 데이터를 분석하고 이상 패턴을 탐지하는 업무에 활용될 수 있습니다.

## 4. 성격의 장단점

### 장점: 끈기 있는 문제 해결

저의 가장 큰 장점은 **어려운 문제 앞에서도 포기하지 않고 끝까지 해결책을 찾는 끈기**입니다.

RF-DETR 프로젝트 초기에는 의료 데이터 부족으로 모델 성능이 기대에 미치지 못했습니다. 단순한 데이터 증강으로는 한계가 있다고 판단하고, 논문을 찾아보며 Elastic Deformation과 Grid Distortion 같은 고급 기법을 학습하고 적용했습니다. 또한 실시간 처리를 위해 Weight Pruning, Query 최적화 등 여러 경량화 기법을 실험하며 정확도와 속도의 최적점을 찾았습니다.

결과적으로 mAP 7% 향상과 22 FPS를 달성하며 금상을 수상할 수 있었습니다. 이 경험은 단순히 수상의 의미를 넘어, **문제를 다각도로 분석하고 체계적으로 접근하는 방법**을 배운 중요한 계기였습니다.

### 단점: 완벽주의 성향

저의 단점은 **과도한 완벽주의로 인해 때때로 세부사항에 집중하느라 전체 일정을 놓칠 수 있다**는 점입니다.

프로젝트 초기에는 모델의 정확도를 0.1%라도 더 높이기 위해 과도하게 시간을 투자하는 경향이 있었습니다. 그러나 캡스톤 프로젝트를 진행하며 팀원들과 협업하는 과정에서, **일정 내에 완성도 있는 결과물을 만드는 것**이 더 중요하다는 것을 배웠습니다.

이후로는 프로젝트 초기에 명확한 마일스톤을 설정하고, 각 단계에서 '충분히 좋은' 수준이 무엇인지 객관적으로 판단하려 노력하고 있습니다. 실시간 시스템에서는 완벽한 알고리즘보다 **안정적이고 실용적인 시스템**이 더 가치 있다는 것을 깨달았고, 이제는 비즈니스 목표와 기술적 완성도의 균형을 맞추는 관점을 가지게 되었습니다.
