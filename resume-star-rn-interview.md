# AI Engineer STAR-RN Interview Answers

## 1. 스마트 안전 관제 시스템

### 30초 버전

CCTV 관제자가 모든 화면을 24시간 집중해서 보기 어렵다는 문제를, AI가 판단 공백을 줄이는 실시간 관제 보조 시스템 문제로 재정의했습니다. RTSP 영상 입력, AI 추론, MQTT 이벤트, WebSocket 알림을 분리하고, frameId/timestamp 기반 evidence chain으로 영상과 이벤트가 맞는 구조를 설계했습니다. 모델 성능뿐 아니라 FP/FN, threshold, tracker state, frame drop 같은 운영 안정성까지 고려한 점이 핵심입니다.

### 1분 버전

스마트 안전 관제 프로젝트에서는 단순히 낙상 탐지 모델을 만드는 것이 아니라, 관제자가 놓칠 수 있는 판단 공백을 AI가 줄이는 실시간 보조 시스템을 목표로 했습니다. 저는 RTSP 영상 입력, YOLO Pose 기반 keypoint 추출, LSTM 기반 시계열 행동 분류, MQTT event metadata, WebSocket 알림이 하나의 흐름으로 이어지는 구조를 정리했습니다. 특히 영상 송출과 AI 이벤트 metadata를 분리하고, frameId/timestamp/evidence chain으로 overlay 동기화와 사후 검증이 가능하도록 보는 관점을 강조했습니다. 아직 일부 구현 세부는 Git 검증이 필요하지만, 이 프로젝트를 통해 AI 모델 정확도만으로는 서비스 품질을 보장할 수 없고, FP/FN 분석과 hard negative 후보 설계가 운영 안정성의 핵심이라는 점을 배웠습니다.

### 2분 버전

스마트 안전 관제 시스템은 CCTV가 많아질수록 관제자가 모든 화면을 동일한 집중도로 확인하기 어렵다는 문제에서 출발했습니다. 저는 이 문제를 "낙상 탐지 모델 구현"이 아니라, "실시간 영상 AI가 관제자의 판단 공백을 줄이고, 그 판단 근거가 영상 증거와 일치해야 하는 관제 보조 시스템" 문제로 재정의했습니다.

제가 정리한 핵심 구조는 RTSP 영상 입력, AI 추론, MQTT 이벤트 발행, WebSocket 대시보드 알림입니다. AI 쪽에서는 YOLO Pose 기반 keypoint 추출과 LSTM 기반 시계열 행동 분류를 핵심 방향으로 두고, 서비스 관점에서는 영상 송출과 AI event metadata를 분리했습니다. 이렇게 해야 frameId, timestamp, cameraId 같은 evidence chain을 남겨 overlay 동기화와 사후 오류 분석이 가능하기 때문입니다.

또한 관제 시스템에서는 모델이 맞췄는지뿐 아니라 왜 틀렸는지가 중요하다고 판단했습니다. 그래서 FP/FN 분석, hard negative 후보, threshold 조정, tracker state, frame drop, active_tracks=0 상태를 운영 안정성 분석 항목으로 정리했습니다. 현재 Recall, Precision, F1 같은 정량 수치는 검증 로그 확보 전이라 이력서에는 임의로 쓰지 않고 있습니다. 다음 단계에서는 실제 Git 저장소와 실험 로그를 확인해 정량 평가와 MQTT payload, frame sync 코드를 보강할 계획입니다.

## 2. LLM Wiki / Evidence Wiki / RAG

### 30초 버전

LLM Wiki 프로젝트는 실험 결과, 오류 해결, 의사결정 기록이 흩어지는 문제를 해결하기 위해 시작했습니다. 문서 기반 Evidence Wiki와 RAG 구조를 사용해 개발 기록을 검색 가능하고 재사용 가능한 지식 자산으로 정리했습니다. 단순 챗봇이 아니라 포트폴리오, 트러블슈팅, 면접 답변의 근거 자료로 활용하는 것이 핵심입니다.

### 1분 버전

프로젝트가 복잡해지면서 실험 결과와 오류 해결 기록, 의사결정 근거가 여러 폴더와 도구에 흩어지는 문제가 있었습니다. 저는 이 문제를 단순 문서 정리가 아니라 개발 지식을 검색 가능하고 재사용 가능한 Evidence Wiki로 전환하는 문제로 봤습니다. RAG 구조를 활용해 문서와 검증 결과, UI 스크린샷, walkthrough를 source evidence로 축적했고, Hybrid Search는 metadata filtering, BM25, vector search, RRF 방향으로 설계했습니다. GraphRAG는 구현 완료가 아니라 향후 확장 후보로만 분리했습니다. 이 경험을 통해 AI 프로젝트에서는 코드뿐 아니라 실패 기록과 의사결정 근거도 중요한 자산이라는 점을 배웠습니다.

### 2분 버전

LLM Wiki / Evidence Wiki 프로젝트는 프로젝트 기록이 흩어지는 문제에서 출발했습니다. AI 프로젝트를 진행하다 보면 모델 선택 이유, 실험 실패, 오류 해결, 검증 결과가 여기저기 남습니다. 시간이 지나면 같은 오류를 다시 찾거나 면접에서 근거 있게 설명하기 어려워집니다.

저는 이 문제를 개발 과정에서 생기는 지식을 검색 가능하고 재사용 가능한 형태로 정리하는 문제로 재정의했습니다. 문서 기반 Wiki와 RAG 구조를 사용해 `implementation_plan.md`, `walkthrough.md`, `task.md`, RAG UI 스크린샷, validation log를 evidence로 축적했습니다. RAG 답변은 source와 함께 제시되도록 설계했고, hallucination 방지를 위해 근거 기반 답변 관점을 강조했습니다.

검색 구조는 Hybrid Search 방향으로 정리했습니다. 정확한 오류 메시지나 파일명은 BM25가 강하고, 의미가 유사한 실험 기록은 vector search가 강하기 때문에 metadata filtering, BM25, vector search, RRF, optional re-ranking 순서로 고도화 계획을 세웠습니다. GraphRAG는 아직 구현 완료로 쓰지 않고, 문서 관계와 규모가 커졌을 때의 확장 후보로만 표현합니다. 이 프로젝트는 제가 AI 도구를 단순히 사용하는 데서 그치지 않고, 개발 지식 관리와 설명 가능성 개선에 연결할 수 있음을 보여줍니다.

## 3. RF-DETR 실시간 대장 내 용종 검출

### 30초 버전

RF-DETR 프로젝트에서는 대장 내시경 영상의 조명 변화와 장벽 왜곡 때문에 일반 객체 탐지만으로는 성능이 흔들리는 문제를 다뤘습니다. Grid Distortion, Elastic Deform으로 데이터 증강을 적용하고 RF-DETR + DINOv2 기반 파인튜닝을 구성했습니다. 기존 자료 기준 mAP 약 7%p 개선, 22 FPS 이상 실시간 추론, 캡스톤 금상과 전국 경진대회 동상을 달성했습니다.

### 1분 버전

대장 내시경 영상은 조명 변화, 반사광, 장벽 왜곡, 병변 형태 다양성 때문에 일반 객체 탐지 모델이 오버피팅되거나 정확도가 흔들리기 쉽습니다. 저는 모델을 무작정 키우기보다 의료 영상의 왜곡은 데이터 단계에서 보완하고, Edge 환경 제약은 모델 경량화로 해결하는 접근을 선택했습니다. Grid Distortion과 Elastic Deform 기반 증강으로 장벽 왜곡과 점막 형태 변화를 반영했고, RF-DETR + DINOv2 기반 추론 구조와 OpenCV 시각화 흐름을 구성했습니다. 기존 자료 기준 mAP 약 7%p 개선과 22 FPS 이상 실시간 추론을 확보했고, 이 경험을 통해 의료 비전에서는 데이터 특성에 맞는 Data-centric 개선이 중요하다는 점을 배웠습니다.

### 2분 버전

RF-DETR 기반 대장 내 용종 검출 프로젝트에서는 의료 영상 특유의 조명 변화, 반사광, 장벽 왜곡, 병변 형태 다양성이 가장 큰 문제였습니다. 초기 접근에서는 일반 객체 탐지 모델을 적용하더라도 데이터 특성 때문에 성능이 안정적으로 나오지 않을 수 있다고 판단했습니다. 그래서 이 문제를 단순 모델 교체가 아니라, 의료 영상 특성을 반영한 Data-centric 객체 탐지 문제로 봤습니다.

저는 Grid Distortion과 Elastic Deform 같은 기하학적 데이터 증강을 적용해 대장 내벽의 왜곡과 점막 형태 변화를 학습 데이터 단계에서 반영했습니다. 모델 구조는 RF-DETR + DINOv2 기반 파인튜닝으로 구성했고, OpenCV 기반 후처리와 시각화 흐름을 통해 실제 영상 입력에서 탐지 결과를 검토했습니다. 또한 Edge 환경에서 실시간성이 중요했기 때문에 Structural Pruning과 22 FPS 이상 추론 성능 확보를 중요한 기준으로 봤습니다.

성과로는 기존 자료 기준 mAP 약 7%p 개선, 22 FPS 이상 실시간 추론, 제17회 건양대학교 캡스톤디자인 경진대회 금상/대상과 전국 공학교육혁신 컨소시엄 동상을 기록했습니다. 이 경험을 통해 의료 AI에서는 모델 구조의 새로움보다 데이터 특성, 후처리, 평가 기준에 맞는 안정적인 개선이 더 중요하다는 점을 배웠습니다.

## 4. VAE 기반 비지도 유방암 병변 검출

### 30초 버전

VAE 프로젝트에서는 유방 초음파 병변 라벨링 비용이 높다는 문제를 정상 조직 기반 비지도 이상 탐지로 재정의했습니다. VAE로 정상 분포를 학습하고 Reconstruction Error Map과 Dynamic Threshold로 병변 후보를 추출했습니다. 로컬 README 기준 max 90% Dice Coefficient와 공학혁신상 수상 기록이 있습니다.

### 1분 버전

유방 초음파 영상은 병변 라벨링 비용이 높아 충분한 지도학습 데이터를 확보하기 어렵습니다. 저는 정상 조직 이미지의 분포를 VAE로 학습하고, 입력 이미지와 재구성 이미지의 차이를 Reconstruction Error Map으로 계산해 병변 후보를 찾는 비지도 이상 탐지 방식으로 접근했습니다. 단순 고정 임계값은 영상별 밝기와 노이즈 차이에 취약했기 때문에 Dynamic Threshold 후처리를 적용했습니다. 로컬 README 기준 max 90% Dice Coefficient를 달성했고, 공학혁신상을 수상했습니다. 이 경험을 통해 의료 AI에서는 모델 구조뿐 아니라 후처리와 평가 기준이 결과 품질을 크게 좌우한다는 점을 배웠습니다.

### 2분 버전

VAE 기반 유방암 병변 검출 프로젝트는 라벨링 비용이 높은 의료 데이터 문제에서 출발했습니다. 유방 초음파 병변을 지도학습으로 검출하려면 전문의가 병변 경계를 정밀하게 주석 처리해야 하는데, 이는 비용과 시간이 많이 들고 데이터 확보도 어렵습니다. 그래서 저는 이 문제를 라벨을 많이 모으는 방향이 아니라 정상 조직 데이터만으로 이상 패턴을 찾는 비지도 이상 탐지 문제로 재정의했습니다.

구현 방향은 VAE를 사용해 정상 조직 이미지의 분포를 학습하는 것이었습니다. 모델이 정상 패턴을 재구성하도록 학습한 뒤, 입력 이미지와 재구성 이미지의 차이를 Reconstruction Error Map으로 계산했습니다. 여기에 KLD와 MSE의 균형을 조절하는 커스텀 손실 함수와, 영상별 밝기/노이즈 차이를 보정하기 위한 Dynamic Threshold 후처리를 적용했습니다.

로컬 README 기준 max 90% Dice Coefficient와 2024 창의혁신 DNA 산학협력 공학혁신상 수상 기록이 있습니다. 이 프로젝트를 통해 AI 모델 성능은 구조만으로 결정되지 않고, 데이터 전처리, 후처리, 평가 기준이 함께 맞아야 실제 결과 품질이 나온다는 점을 배웠습니다.
